## 공통 SEO/페이지 규칙
- 각 서비스는 **MDX 프론트매터**를 사용 (아래 템플릿 복붙).
- `keywords`는 사용하지 않습니다. `description`, `canonical`, `ogImage`만 채우세요.
- 모든 페이지에 **JSON‑LD(WebPage)** 1종 이상 포함. 필요 시 `Product/SoftwareApplication/MedicalWebPage` 등 추가.
- CTA는 최대 2개(Primary/Secondary) + 내부 링크(문서/지원/가격 등) 3개 이내.
- 이미지 1장(1200×630, &lt;150KB), WebP 권장.

### MDX 프론트매터 공통 템플릿
```md
---
slug: "<서비스슬러그>"
title: "<서비스명 — 한 줄 포지셔닝>"
subtitle: "<핵심 가치 요약>"
updated: "2025-09-22"
audience: ["주요 타깃A", "주요 타깃B"]
cta:
  primary: { label: "<주요 CTA>", href: "https://..." }
  secondary: { label: "<보조 CTA>", href: "https://..." }
seo:
  description: "<110–150자 설명>"
  canonical: "https://dreamailab.com/services/<slug>"
  ogImage: "/og/<slug>.jpg"
schema:
  webpage_name: "<탭/OG 제목>"
  datePublished: "2023-01-01"
  dateModified: "2025-09-22"
---
```
