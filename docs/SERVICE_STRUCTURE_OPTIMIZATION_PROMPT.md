# DreamAILab Service Structure Optimization Prompt

## Objective
We are restructuring DreamAILab service ecosystem pages to reduce information overload, clarify service positioning, and standardize page structure across all services.

**This phase is structural optimization, not a redesign.**

## Execution Scope
- **This step focuses only on structural alignment.**
- Do NOT rewrite service content yet.
- Do NOT redesign UI.
- Do NOT change service messaging.
- **We are only reorganizing page layout.**

## Core Goals
- Standardize service page layout
- Improve readability
- Clarify service status
- Strengthen DAL ecosystem positioning
- Prepare pages for future scalable updates

## Required Service Page Structure
All service pages must follow this order:

1. **Hero Section** — One-line core value explaining what the service does and for whom.
2. **Problem Definition** — Describe user or market pain points.
3. **Solution** — Explain how the service solves the problem.
4. **User Workflow** — Show how users move from entry to outcome.
5. **Technology Advantage (Short)** — Brief technical advantage only. No deep technical explanations.
6. **Business Impact** — Impact on users, institutions, or markets.
7. **Service Status** — Must clearly display one: Live Service | Beta | Pilot | In Development
8. **Ecosystem Connection** — Show how the service connects to DAL platform and other services.
9. **CTA** — Service access, inquiry, or partnership link.

## Technical Content Rule
**Remove or relocate** deep technical explanations:
- AI architecture
- model explanations
- reinforcement learning details
- data pipeline details

→ Move these to Technology or Whitepaper pages.

## NoahAI Handling
Mark NoahAI as:
- migrated to Noah AI Labs
- formerly part of DAL ecosystem
- development updates external
- separate legal entity

## Mobile Optimization Rule
Restructure pages to:
- shorten paragraphs
- reduce text density
- improve section separation
- reduce scroll fatigue

## Execution Principle
- This phase is **NOT** redesign
- This phase is **NOT** UI change
- This phase is **ONLY** structure optimization
- This phase is **ONLY** information organization

## Final Focus
Clarity, scalability, ecosystem consistency.
