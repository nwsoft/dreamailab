## 🚀 1회 설정 (최초 1회만)

### 1-1. GitHub 저장소 준비

- GitHub에서 `nwsoft/dreamailab` 저장소가 이미 생성되어 있어야 합니다.
- 저장소가 없다면: [GitHub](https://github.com/new) → 새 저장소 생성 → 이름 `dreamailab`, Public 선택

### 1-2. 로컬에서 GitHub 연결 및 푸시

```bash
# 스크립트 실행 (프로젝트 루트에서)
chmod +x scripts/setup-github-deploy.sh
./scripts/setup-github-deploy.sh

# 커밋 메시지 지정 (선택)
./scripts/setup-github-deploy.sh "메시지: DAL 플랫폼 메시지 구조 정렬"
```

또는 수동 실행:

```bash
git init
git add .
git commit -m "Initial commit: Dream AI Lab website"
git branch -M main
git remote add origin https://github.com/nwsoft/dreamailab.git
git push -u origin main
```

### 1-3. Cloudflare Pages ↔ GitHub 연결

1. [Cloudflare Dashboard](https://dash.cloudflare.com) → **Workers & Pages**
2. **dreamailab** 프로젝트 선택 (또는 **Create application** → **Pages** → **Connect to Git**)
3. **Connect to Git** → **GitHub** 선택
4. `nwsoft/dreamailab` 저장소 선택 후 **Begin setup**

**빌드 설정 (Pages):**

| 설정 항목 | 값 |
|-----------|-----|
| Framework preset | Next.js (Static HTML Export) 또는 None |
| Build command | `npm run build` |
| Build output directory | `out` |
| Root directory | (비워두기) |
| Deploy command | (비워두기 — Pages는 자동 배포) |

**Workers 프로젝트인 경우:**

| 설정 항목 | 값 |
|-----------|-----|
| Build command | `npm run build` |
| Deploy command | `npx wrangler deploy` |
| Root directory | (비워두기) |

5. **Save and Deploy** 클릭

## 📬 문의 폼 (Formspree, 선택)

정적 export 사이트이므로 문의는 **Formspree** 또는 **mailto** 로 처리합니다.

| 모드 | 조건 | 동작 |
|------|------|------|
| **Formspree** | `NEXT_PUBLIC_FORMSPREE_FORM_ID` 설정됨 | 제출 시 Formspree로 JSON POST → 이메일 수신 |
| **mailto** | 환경 변수 없음 | 메일 앱에 제목·본문 prefill (사용자가 최종 발송) |

### 설정 절차

1. [Formspree](https://formspree.io) — **이미 폼이 있으면** 새로 만들 필요 없음. 로그인 → 해당 폼 → **Integration** 또는 폼 URL에서 ID 확인 (`https://formspree.io/f/여기가ID`)
2. Formspree 폼 설정 → **Notifications** 수신 이메일: `contact@dreamailab.com` (Cloudflare 이메일 라우팅과 별개로 Formspree에도 지정)
3. Cloudflare → **Workers 및 Pages** → **dreamailab** → **설정** → **변수 및 암호**
4. **변수 추가** (이름·값 칸만 보이면 **일반 변수**로 추가 — **암호(Secret) 아님**)
   - **이름:** `NEXT_PUBLIC_FORMSPREE_FORM_ID`
   - **값:** Form ID만 (예: `xyzabcde`) — 전체 URL도 가능
   - **암호를 쓰지 않는 이유:** `NEXT_PUBLIC_` 값은 빌드 후 사이트 JS에 그대로 들어가 공개됩니다. Form ID는 Formspree가 공개 엔드포인트로 쓰는 값이라 **변수(Plain text)** 로 충분합니다.
   - **환경:** Production (Preview도 쓰면 동일 값)
5. **배포** 탭 → 최신 프로덕션 **다시 배포** (변수만 저장하면 반영 안 됨 — Next.js 정적 빌드 시 박힘)
6. `https://dreamailab.com/contact` 테스트 → 버튼이 **「문의 보내기」** 이면 Formspree ON

**Formspree 로그인이 뭔가요?**  
formspree.io에 가입할 때 쓴 **이메일·Google 계정**입니다. 폼을 만든 사람/팀 계정으로 로그인하면 기존 폼이 보입니다.

로컬 개발: 프로젝트 루트에 `.env.local` 생성 (`.env.example` 참고). Git에는 커밋하지 않습니다.

```bash
# .env.local (예시)
NEXT_PUBLIC_FORMSPREE_FORM_ID=your_form_id_here
```
