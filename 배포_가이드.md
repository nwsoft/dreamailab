## 🚀 1회 설정 (최초 1회만)

### 1-1. GitHub 저장소 준비

- GitHub에서 `nwsoft/dreamailab` 저장소가 이미 생성되어 있어야 합니다.
- 저장소가 없다면: [GitHub](https://github.com/new) → 새 저장소 생성 → 이름 `dreamailab`, Public 선택

### 1-2. 로컬에서 GitHub 연결 및 푸시

```bash
# 스크립트 실행 (프로젝트 루트에서)
chmod +x scripts/setup-github-deploy.sh
./scripts/setup-github-deploy.sh

# 커밋 메시지 지정 (선택)
./scripts/setup-github-deploy.sh "메시지: DAL 플랫폼 메시지 구조 정렬"
```

또는 수동 실행:

```bash
git init
git add .
git commit -m "Initial commit: Dream AI Lab website"
git branch -M main
git remote add origin https://github.com/nwsoft/dreamailab.git
git push -u origin main
```

### 1-3. Cloudflare Pages ↔ GitHub 연결

1. [Cloudflare Dashboard](https://dash.cloudflare.com) → **Workers & Pages**
2. **dreamailab** 프로젝트 선택 (또는 **Create application** → **Pages** → **Connect to Git**)
3. **Connect to Git** → **GitHub** 선택
4. `nwsoft/dreamailab` 저장소 선택 후 **Begin setup**

**빌드 설정 (Pages):**

| 설정 항목 | 값 |
|-----------|-----|
| Framework preset | Next.js (Static HTML Export) 또는 None |
| Build command | `npm run build` |
| Build output directory | `out` |
| Root directory | (비워두기) |
| Deploy command | (비워두기 — Pages는 자동 배포) |

**Workers 프로젝트인 경우:**

| 설정 항목 | 값 |
|-----------|-----|
| Build command | `npm run build` |
| Deploy command | `npx wrangler deploy` |
| Root directory | (비워두기) |

5. **Save and Deploy** 클릭

## 🔄 일상적인 업데이트 배포

코드 수정 후 GitHub에 푸시하면 **Cloudflare Pages가 자동으로 빌드·배포**합니다.

```bash
git add .
git commit -m "메시지: 변경 내용 요약"
git push origin main
```

푸시 후 1~3분 내에 dreamailab.com에 반영됩니다.

## 📬 문의 폼 (mailto)

`/contact` 제출 시 **메일 앱**이 열리고, `contact@dreamailab.com` 으로 제목·본문이 채워집니다. 방문자가 **보내기**를 눌러야 전달됩니다.

- **수신:** Cloudflare **이메일 라우팅**으로 `contact@dreamailab.com` → 실제 받은편지함
- **추가 설정 불필요:** Formspree·Pages 환경 변수 없음
- **코드:** `app/contact/page.tsx`, `lib/contact-form.ts`

## ⚠️ 주의사항

- **node_modules, .next, out** → `.gitignore`로 제외됨 (업로드 안 됨)
- **main** 브랜치가 Cloudflare Pages 기본 배포 브랜치로 설정되는 것이 일반적입니다.
- 배포 상태: Cloudflare Dashboard → Workers & Pages → dreamailab → **Deployments** 탭에서 확인
